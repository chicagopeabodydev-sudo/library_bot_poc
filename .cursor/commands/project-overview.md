# project-overview

## Overview
This project is a simple chatbot uses retrieval augmented generation (RAG) when the chatbot answers user questions.

## RAG steps
The content available to the RAG process is generated by a two-step process:
-- 1. crawl4ai is used to connect to a website and scrape its webpages and generates markdown files for these webpages (note: crawl4ai gets the website's home page URL from an environment variable)
-- 2. The markdown files created by crawl4ai are indexed by LlamaIndex using an embeddings model and the results are stored in a pgvector collection hosted on Supabase

## Chatbot steps
Once the crawl, indexing, and embeddings storage are complete, chatbot UI developed with Streamlit is used to handle user questions. The steps to answering the user questions include applying guardrails at multiple points in the process to make the application safer and more secure by blocking inappropriate, off-topic, or malicious user inputs or LLM responses. Nemo-guardrails are used to apply guardrails.

The chatbot steps are:
-- 1. User enters a question (which is plain text) into the Streamlit UI
-- 2. The questions is checked against all congfigured input guardrails to verify it is safe and on topic (meaning, related to the website content scraped and processed in the first phase)
-- 3. IF the question FAILS any input guardrails, then an appropriate "error" messsage is returned to the user via the Streamlit interface
-- 4. OR, if the questions PASSES all guardrails, then LLamaIndex pulls the appropriate content from the pgvector storage and passes this along with the question to an LLM to get a response
-- 5. After the response is returned by the LLM, it is checked against all configured output guardrails (also using Nemo-guardrails)
-- 6. IF the response FAILS any output guardrails, then an "error" message is returned to the user via the Streamlit interface
-- 7. OR, if the response PASSES all output guardrails, then the response is returned to the user via the Streamlit interface
